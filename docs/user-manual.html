b2ctl · disk lifecycle tool · คู่มือผู้ใช้

ดูแลดิสก์ ZFS ด้วย b2ctl
ฉบับ "ไม่เคยใช้มาก่อนก็ทำตามได้"

เครื่องมือ command-line ดูสุขภาพดิสก์และจัดการ ZFS pool บน Dell R620 (Proxmox VE 9.2) — คู่มือนี้พาใช้ทีละขั้น พร้อมตัวอย่างหน้าจอจริง

Server: Dell R620 OS: Proxmox VE 9.2 Mode: IT / HBA Storage: ZFS Pools: rpool + tank
อ่านคู่มือนี้ยังไงดี? ถ้าเพิ่งเริ่ม — อ่านหัวข้อ 00 → 05 ตามลำดับก่อนหนึ่งรอบ จะเห็นภาพรวมทั้งหมด. เวลาเจอปัญหาหน้างาน (ดิสก์เสีย, ต้องเปลี่ยนดิสก์) ให้กระโดดไปที่ หัวข้อ 07 — Runbook ได้เลย. ทุกคำสั่งที่ "แตะดิสก์จริง" b2ctl จะถามยืนยันก่อนเสมอ — กด Enter เปล่า ๆ = ยกเลิก ปลอดภัย.
00

b2ctl คืออะไร?

b2ctl คือเครื่องมือสำหรับ ดูแลดิสก์ บนเซิร์ฟเวอร์ Dell R620 ที่ใช้การ์ด RAID (PERC H710) ถูกแปลง (crossflash) เป็นโหมด IT/HBA — คือให้ระบบเห็นดิสก์ดิบ ๆ ตรง ๆ แล้วใช้ ZFS จัดการ pool แทนตัวการ์ด RAID

เปรียบเทียบง่าย ๆ: b2ctl เหมือน "หมอประจำตัว" ของดิสก์ในเครื่อง — ตรวจสุขภาพ, บอกว่าตัวไหนใกล้เสีย, แนะนำการรักษา (เปลี่ยนดิสก์ / เพิ่มตัวสำรอง) แล้วช่วยทำให้เสร็จ โดยคุณไม่ต้องจำคำสั่ง ZFS ยาว ๆ เอง

b2ctl ทำอะไรได้บ้าง

01ดูสุขภาพดิสก์

ตารางรวมทุกดิสก์ — รุ่น, ซีเรียล, ชั่วโมงใช้งาน, ความสึกหรอ, อายุที่เหลือ, bad sector และ "เกรดรวม" (LEVEL) สีเดียวรู้เรื่อง

02จัดการ ZFS pool

เพิ่มดิสก์สำรอง (spare), สลับดิสก์, เปลี่ยนดิสก์เสีย, สร้าง/ลบ pool — ผ่านเมนูกดตัวเลข ไม่ต้องพิมพ์ zpool เอง

03หาตัวดิสก์จริง

กะพริบไฟ LED ที่ช่องของดิสก์ตัวนั้น เพื่อให้แน่ใจว่าดึงถูกตัวก่อนถอด

04เฝ้าดูอัตโนมัติ

เปิดโหมด watch ทิ้งไว้ — เสียบหรือถอดดิสก์เมื่อไหร่ ระบบแจ้งเตือนทันทีใน 2–3 วินาที

ศัพท์ที่ควรรู้ก่อน (แค่ 5 คำ)

คำแปลว่า
ZFSระบบไฟล์ + ตัวจัดการ RAID แบบซอฟต์แวร์ ที่ b2ctl ควบคุมให้
poolกลุ่มดิสก์ที่รวมกันเป็นที่เก็บข้อมูลก้อนเดียว — เครื่องนี้มี rpool (ดิสก์บูต) และ tank (ข้อมูล)
vdev"กลุ่มย่อย" ภายใน pool เช่น mirror-0, raidz1-0 — คือรูปแบบการวางดิสก์ให้ทนเสีย
spareดิสก์สำรอง เสียบพร้อมใช้ ถ้ามีตัวไหนเสีย ZFS จะดึงมันเข้าไปแทนอัตโนมัติ
resilverการ "สร้างข้อมูลใหม่" ลงดิสก์ที่เพิ่งเปลี่ยน — เหมือน rebuild ของ RAID เดิม
เครื่องนี้มี 2 pool: rpool = mirror (RAID1) ดิสก์บูต Proxmox · tank = raidz1 (RAID5) ดิสก์ข้อมูล 3 ลูก + สำรอง 1 ลูก. raidz1 ทนดิสก์เสียได้ 1 ลูก — เสียลูกที่สองก่อน resilver เสร็จ = ข้อมูลหาย จึงต้องรีบเปลี่ยน
01

การติดตั้ง

ทำครั้งเดียวตอนตั้งเครื่อง บน Proxmox ให้รันในฐานะ root (ไม่ต้องมี sudo)

root@pve:~# git clone https://github.com/JPcGaM3/b2ctl-itmode.git b2ctl
root@pve:~# cd b2ctl/codes
root@pve:~# ./install.sh

./install.sh เปล่า ๆ = ลง เฉพาะตัว b2ctl (โปรแกรม + launcher) ไม่โหลด tool เสริม ไม่แตะ apt ไม่ต้องต่อเน็ต. เลือกรูปแบบเพิ่มเติมได้ตามฮาร์ดแวร์:

คำสั่งลงอะไร
./install.shเฉพาะ b2ctl — ไม่มี tool เสริม (ใช้ได้เลยถ้าเครื่องมี smartctl + zpool อยู่แล้ว)
./install.sh --with-toolsb2ctl + tool ทั้งคู่ (sas2ircu + perccli) โหลดจาก Google Drive
./install.sh --flashเครื่อง IT/HBA (crossflashed): ลง sas2ircu + ตั้งโหมดเป็น it ← เครื่องนี้
./install.sh --percเครื่อง Dell PERC RAID จริง: ลง perccli + ตั้งโหมดเป็น raid

เครื่องต้องมีอะไรก่อน

โปรแกรมทำหน้าที่จำเป็น?
smartctlอ่านข้อมูลสุขภาพดิสก์ (SMART)จำเป็น
zpoolจัดการ ZFS poolจำเป็น
sas2ircuบอกหมายเลขช่อง (bay) ของดิสก์มีก็ดี
ไม่มี sas2ircu ก็ใช้ได้ — แค่คอลัมน์ BAY จะเป็น - (ไม่รู้เลขช่อง). ทุกอย่างอื่นทำงานปกติ เพราะ b2ctl ยึด ซีเรียล ของดิสก์เป็นหลัก ไม่ใช่เลขช่อง
02

ตรวจสภาพระบบก่อนใช้

ใช้ครั้งแรกหลังติดตั้ง หรือเวลาสงสัยว่า tool ครบไหม / backend ถูกไหม — คำสั่ง b2ctl check

root@pve:~# b2ctl check

[b2ctl environment check]
  [✔] Running as root
  [✔] smartctl     /usr/sbin/smartctl       (smartctl 7.5 ...)
  [✔] sas2ircu     /usr/sbin/sas2ircu       (LSI SAS2 IR Config Utility)
  [✔] zpool        /usr/sbin/zpool          (zfs-2.4.2-pve1)
  [✔] wipefs       /usr/sbin/wipefs         (util-linux 2.41)
  [✔] sgdisk       /usr/sbin/sgdisk         (sgdisk 1.0.10)

  [✔] Detected backend: IT-mode
  [✔] Controllers found: 6 (6 disks in bay map)
  [!] Config: /etc/b2ctl/config.json (missing — using defaults)
อ่านผลยังไง
  • sas2ircu [✔] สำคัญสุด — ถ้าขึ้น [✗] หรือ "won't execute" คอลัมน์ BAY จะเป็น - ทั้งหมด. แก้ด้วย apt-get install -y libc6-i386 (sas2ircu เป็นไฟล์ 32-bit)
  • backend = IT-mode = ถูกต้องสำหรับเครื่อง crossflash นี้
  • Controllers found: 6 — เลขดิสก์ > 0 แปลว่าอ่านเจอดิสก์และ bay map ทำงาน
  • Config ... missing ไม่ใช่ error — b2ctl ใช้ค่า default ได้ทันที ไม่ต้องสร้าง config เอง

อ่านตารางให้เป็น

ถ้าอ่านตารางนี้ออก คุณใช้ b2ctl ได้ 80% แล้ว — แต่ละคอลัมน์แปลว่า:

คอลัมน์ความหมายตัวอย่าง
BAYหมายเลขช่อง (ตู้:ช่อง)1:4 = ตู้ 1 ช่อง 4
DEVชื่อ device ใน Linuxsda, sdb
IFชนิดการเชื่อมต่อSAS, SATA, NVMe
MODELรุ่นดิสก์Samsung SSD 870
SERIALซีเรียล (ตัวจริงที่ b2ctl ใช้อ้างอิง)S74ZNS0W…
POWER_ONชั่วโมงเปิดใช้งานสะสม18281h(~2.1y)
WEARสึกหรอไปกี่ % (ยิ่งน้อยยิ่งดี)1%
ENDอายุที่เหลือ (คำนวณจาก TBW)98.4%
WRITTENเขียนไปแล้ว / TBW ที่รับรอง9.7TB/600TBW

LEVELเกรดรวม (ดูด้านล่าง)—

LEVEL — เกรดรวม ดูสีเดียวรู้เรื่อง

เกรดแปลว่าต้องทำอะไร
NORMALดิสก์แข็งแรง อยู่ใน pool เรียบร้อยไม่ต้องทำอะไร

06

คำสั่งทีละปุ่ม — กดแล้วเจออะไร

แต่ละปุ่มในโหมด watch อธิบายแบบ "กดอะไร → เห็นอะไร → แปลว่าอะไร" ปุ่มสีน้ำเงินคือปุ่ม อ่านอย่างเดียว (ปลอดภัย ไม่แตะดิสก์) ปุ่มสีเขียวคือปุ่มที่ เปลี่ยนแปลงดิสก์ (จะถามยืนยันก่อนเสมอ)

r

Refresh รีเฟรชตาราง

สแกนดิสก์ใหม่แล้วแสดงตารางอีกครั้ง — ใช้ตอนเพิ่งเสียบดิสก์หรือ pool health ยังไม่อัปเดต กดได้เรื่อย ๆ

b2ctl> r
# สแกนใหม่ แล้วพิมพ์ตาราง + Pools + summary อีกครั้ง (เหมือน b2ctl status)
a

Assign จัดสรรดิสก์ว่างเข้า pool

ใช้กับดิสก์ที่ขึ้น CONFIG (อยู่ในเครื่องแต่ยังไม่ได้อยู่ pool ไหน) → เลือกดิสก์ → เด้งเมนูให้ทำอะไรกับมัน

ขั้นที่ 1 — a จะลิสต์เฉพาะดิสก์ที่ยังไม่ได้ assign ให้เลือก:

b2ctl> a
    [1] bay 1:7 /dev/sde (Samsung SSD 870, SN S74ZNS0WXXXXXXX)
  assign which #> 1

รายการนี้รวมดิสก์ว่าง 3 แบบ: ดิสก์ว่างปกติ (เปิดเมนู action ด้านล่าง); ดิสก์ [GHOST] (OS ปฏิเสธ ไม่มี /dev, ขึ้นว่า — needs wipe) → ไปทาง wipe/rescue (ดูปุ่ม u); และดิสก์ PERC Unconfigured-Good (เฉพาะเครื่อง RAID mode) → ไปเมนู hardware RAID (set JBOD / สร้าง volume / เพิ่มเป็น spare)

ขั้นที่ 2 — เลือกดิสก์ว่างปกติ แล้วเด้งเมนูเดียวกับตอนเสียบดิสก์ใหม่ (NEW DISK DETECTED):

  Disk /dev/disk/by-id/wwn-0x5002538... is free.
  What do you want to do with it?
    [1] Prepare for physical removal (Blink LED)
    [2] Add to a pool as hot SPARE
    [3] REPLACE a degraded/faulted disk in a pool
    [4] ATTACH to an existing disk (convert to/expand mirror)
    [5] ADD single disk to a pool (expand capacity - WARNING: no redundancy)
    [6] WIPE it blank (for a new pool)
    [s] skip / decide later
  action>
เลือกทำอะไรใช้เมื่อ
[1]กะพริบ LEDอยากรู้ว่าดิสก์อยู่ช่องไหนก่อนดึงออก
[2]เพิ่มเป็น SPAREเพิ่มดิสก์สำรองไว้ใน pool
[3]REPLACE ดิสก์เสียมีดิสก์ FAULTED/DEGRADED และจะแทนที่
[4]ATTACH เป็น mirrorอยากเพิ่มความปลอดภัย (ทำ mirror)
[5]ADD เดี่ยวอันตราย เสีย 1 = ข้อมูลหายหมด
[6]WIPE ล้างลบทุกอย่าง เตรียมสร้าง pool ใหม่
[s]ข้ามยังไม่ตัดสินใจ (ปลอดภัยสุด)

ตัวอย่าง — เลือก [2] เพิ่มเป็น spare:

  action> 2
    [1] rpool (ONLINE)
    [2] tank (ONLINE)
  pool #> 2
  add (1:7) Samsung SSD 870 (S74ZNS0WXXXXXXX) to 'tank' as spare? [y/N]> y
  ✔ added as spare
o

Offload ถอดดิสก์ออกจาก pool

เอาดิสก์ออกจาก pool เพื่อถอดออกทางกายภาพ — ระบบทำต่างกันตามชนิดดิสก์

ดิสก์เป็นอะไรระบบทำอะไร
Spare (สำรอง)ถอดออกทันที ไม่ต้อง resilver
Mirror memberถอดออกทันที (ถ้ายังมี leg อื่น ONLINE)
RAIDZ memberต้อง resilver ไปยัง spare ก่อน — ใช้เวลา

ขั้นที่ 1 — o ลิสต์ดิสก์ทุกตัวใน pool ให้เลือก:

b2ctl> o
    [1] bay 1:0 /dev/sdf in rpool (vdev mirror-0)
    [3] bay 1:4 /dev/sdb in tank (vdev raidz1-0)
    [6] bay 1:7 /dev/sde in tank (vdev spares)
  offload which #>

ตัวอย่าง ก — ถอด spare ออก (ทันที ไม่ต้อง resilver):

  offload which #> 6
  This disk is a hot spare. Remove (1:7) Samsung SSD 870 (…) from 'tank'? [y/N]> y
  ✔ removed from pool

ตัวอย่าง ข — ถอดสมาชิก RAIDZ (ต้อง resilver ไป spare ก่อน):

  offload which #> 3
  Replace (1:4) Samsung SSD 870 (…) onto spare (1:7) Samsung SSD 870 (…)? [y/N]> y
  ✔ replace started — resilvering onto spare
  resilvering... 45.2% done, ETA 00:03:21
  ✔ resilver completed 100%
  ✔ detached old disk /dev/sdb
  please pull bay 1:4 ... blinking LED
s

Swap สลับดิสก์สึกหรอไปยัง spare

ดิสก์ยังไม่เสียแต่เริ่มเก่า (WEAR สูง, END น้อย) — สลับไป spare ก่อนพัง. ต่างจาก offload: swap = สลับที่กัน ดิสก์เก่ากลายเป็น spare อัตโนมัติ ทั้งสองตัวยังอยู่ในเครื่อง

b2ctl> s
  swap which #> 6
  swap (1:7) Samsung SSD 870 (…) onto spare (1:4) …? [y/N]> y
  ✔ swap started — resilvering onto spare
  ✔ resilver completed 100%
  ✔ detached old disk /dev/sde
  ✔ (1:7) Samsung SSD 870 is now a hot spare in 'tank'
d

Demote ลด mirror leg ลงเป็น spare

ใช้กับ mirror ที่มีมากกว่า 2 ขา (เช่น 3-way) ถอด 1 ขาลงไปเป็น spare. มีตัวกันพลาด — ถ้าถอดแล้ว mirror เหลือขาเดียว (ไม่มี redundancy) ระบบจะปฏิเสธ

b2ctl> d
    [1] (1:0) SAMSUNG MZ7LH1T9 (S4F2NY0XXXXXXX) in rpool
    [2] (1:1) SAMSUNG MZ7LH1T9 (S4F2NY0XXXXXXX) in rpool
  demote which #> 2
  demote (1:1) SAMSUNG MZ7LH1T9 (…) in 'rpool' to a hot spare? [y/N]> y
  ✔ demoted to spare
n

New-pool สร้าง pool ใหม่

มีดิสก์ว่างหลายตัว อยากรวมเป็น pool ใหม่ — เลือกดิสก์ (คั่นเว้นวรรค เช่น 1 2 3) → ตั้งชื่อ → เลือกชนิด RAID

b2ctl> n
    [1] /dev/sdb (bay 1:4)
    [2] /dev/sdc (bay 1:5)
    [3] /dev/sdd (bay 1:6)
  pick disks (space-separated #)> 1 2 3
  pool name> backup
  raid type (stripe, mirror, raid10, raidz1, raidz2) [mirror]> raidz1
  create pool 'backup' (raidz1) with 3 disks? [y/N]> y
  ✔ pool created
ชนิดดิสก์ขั้นต่ำทนเสียเนื้อที่ได้
stripe1ไม่มี100%
mirror21 ตัว50%
raidz13+1 ตัว(N-1)/N
raidz242 ตัว(N-2)/N
raid104 (คู่)1 ต่อคู่50%
ถ้าดิสก์มีข้อมูลเก่า ระบบจะเตือนและถามว่าจะ wipe ก่อนไหม. หลังเลือกชนิด RAID b2ctl จะถามค่า property ทีละตัว (ashift, compression ฯลฯ) — กด Enter เพื่อใช้ค่า default ที่เหมาะกับ SSD อยู่แล้ว
e

Extend เพิ่ม cache / log ให้ pool

เร่งความเร็ว pool เดิม: L2ARC cache = เร่งการอ่าน (เสียแล้วแค่ cache miss ไม่กระทบข้อมูล) · SLOG log = เร่งการเขียนแบบ sync (ควรใช้ 2 ลูกทำ mirror + ต้องเป็น SSD ที่มี PLP)

b2ctl> e
  [1] add L2ARC cache (read cache; loss = harmless)
  [2] add SLOG log   (sync-write accel; mirror + PLP recommended)
  [3] remove a cache/log device
  action> 2
    [1] /dev/sdg (bay 1:8)
    [2] /dev/sdh (bay 1:9)
  pick disk(s) (space-separated #)> 1 2
  [!] ensure the SSD(s) have Power-Loss Protection (PLP).
  add SLOG (mirror) to 'tank'? [y/N]> y
  ✔ SLOG added
b

Burn-in ทดสอบดิสก์ก่อนใช้

วนทดสอบดิสก์ใหม่/มือสองก่อนเชื่อใจ — เลือกหลายลูกพร้อมกันได้ รันเบื้องหลัง (รายละเอียดเต็มดูหัวข้อ 08)

b2ctl> b
    [1] /dev/sdb (bay 32:4) Samsung SSD 870 EVO 1TB
    [2] /dev/sda (bay 32:5) Samsung SSD 870 EVO 1TB
  burn in which #> (space-separated) 1 2
  burn-in 2 disk(s) (long self-test)? [y/N]> y
  also run a full read-surface scan (badblocks, read-only, hours)? [y/N]> y
# เปิดหน้าจอสดแสดงแถบความคืบหน้าของแต่ละลูก — Ctrl-C ออกได้ งานยังรันต่อ
t

Toggle dry-run สลับโหมดซ้อม

เปิด/ปิดโหมด "ลองโดยไม่ทำจริง" — คำสั่งเขียนจะแค่แสดงไม่รันจริง (รายละเอียดดูหัวข้อ 09)

b2ctl> t

b2ctl> t

l

Locate หาตัวดิสก์

กะพริบไฟ LED ของดิสก์ตัวที่ระบุ ~5 วินาที (รายละเอียดดูหัวข้อ 11)

b2ctl> l

  blinking /dev/sdc for 5s ...
  ✔ done (via dd)
u

Udev-rescue กู้ดิสก์ที่ OS ปฏิเสธ

ดิสก์เสียบอยู่จริงแต่ OS ไม่รับ (ขึ้นเป็น GHOST ไม่มี /dev) — สั่ง udevadm trigger/settle ให้ kernel มองเห็น เป็นงานอ่านอย่างเดียว/diagnostic ไม่แตะข้อมูลในดิสก์

b2ctl> u
    ghost bay 1:4 serial S74ZNS0WXXXXXXX

x

อันตรายสุด ลบ pool ทั้งก้อนด้วย zpool destroy — ข้อมูลหายหมด ต้องยืนยัน และพิมพ์ชื่อ pool จึงจะทำ (b2ctl ลบ cron ของ pool นั้นให้ด้วย)

b2ctl> x

    [2] tank (2.72T, ONLINE)

07

Runbook — วิธีแก้ปัญหาหน้างานจริง

เจอสถานการณ์จริงแล้วทำตามได้เลย ทีละขั้น

📌 สถานการณ์ 1 — ดิสก์เสีย ต้องเปลี่ยนใหม่

อาการ: ตารางขึ้น LEVEL = CRITICAL, HEALTH = FAILED, หรือ BAD > 0

  1. หาดิสก์ที่เสีย — กด l แล้วพิมพ์ซีเรียลของดิสก์เสีย → ไฟกะพริบที่ช่องนั้น
  2. ดึงดิสก์เสียออก — ถอดจากช่อง (ระบบแจ้ง ■ disk removed)
  3. เสียบดิสก์ใหม่ — ระบบแจ้ง NEW DISK DETECTED พร้อมเมนู
  4. เลือก [3] REPLACE เพื่อแทนที่ดิสก์เสีย แล้วยืนยัน y
  5. รอ resilver เสร็จ — ZFS สร้างข้อมูลใหม่อัตโนมัติ ห้ามปิดเครื่อง/ถอดดิสก์ระหว่างนี้
  action> 3
    [1] tank: /dev/disk/by-id/wwn-0x500... (FAULTED)
  replace #> 1
  replace … in 'tank' with (1:5) Samsung SSD 870 (S74ZNS0W...)? [y/N]> y
  ✔ replace started — resilvering

📌 สถานการณ์ 2 — ดิสก์เริ่มเก่า สลับไป spare ก่อนพัง

อาการ: LEVEL = WARNING, END(left) ต่ำลง, WEAR สูงขึ้น

  1. ดูว่ามี spare ใน pool ไหม (คอลัมน์ POOL มีดิสก์ที่ขึ้น tank/spares)
  2. กด s (swap) → เลือกดิสก์ที่เริ่มเก่า
  3. ยืนยัน y → ระบบ resilver ไป spare แล้วสลับที่กัน
  4. เสร็จ — ดิสก์เก่ากลายเป็น spare, ดิสก์ใหม่ทำงานแทน (ไม่ต้องถอดออก)

📌 สถานการณ์ 3 — เพิ่ม spare ใหม่เข้า pool

  1. เสียบดิสก์ใหม่ → ระบบแจ้ง NEW DISK DETECTED
  2. เลือก [2] Add to a pool as hot SPARE
  3. เลือก pool ที่จะเพิ่ม แล้วยืนยัน y
  action> 2
    [1] rpool (ONLINE)
    [2] tank (ONLINE)
  pool #> 2
  add (1:7) Samsung SSD 870 (…) to 'tank' as spare? [y/N]> y
  ✔ added as spare

📌 สถานการณ์ 4 — ถอดดิสก์ออกจาก pool อย่างปลอดภัย

  1. กด o (offload) → เลือกดิสก์ที่จะถอด
  2. ยืนยัน y — b2ctl offline ดิสก์และให้ spare รับหน้าที่
  3. ระบบกะพริบ LED ช่องนั้น บอกว่าต้องดึงช่องไหน
  4. ดึงดิสก์ออก — ตอนนี้ปลอดภัยแล้ว

📌 สถานการณ์ 5 — ดิสก์กำลังจะเสีย แต่ไม่มี spare และช่องเต็มหมด

ใช้ o (offload) กับดิสก์ตัวนั้น — b2ctl จะ:

  1. เช็กก่อนว่า pool redundant เต็ม ตอนนี้ (ลูกอื่นปกติหมด) ถ้าไม่ครบ → ปฏิเสธ (กันไม่ให้ pool ล่ม)
  2. รัน zpool offline — pool เป็น DEGRADED แต่ยังออนไลน์ + เปิดไฟ LED ช่องนั้น
  3. ถอดดิสก์ช่องนั้น ใส่ดิสก์ใหม่ในช่องเดิม แล้วกด Enter
  4. b2ctl zpool replace ดิสก์ใหม่ + โชว์ความคืบหน้า resilver → เสร็จ pool กลับมา ONLINE
ระวัง: ระหว่าง DEGRADED / resilver ไม่มี redundancy — ถ้าดิสก์ลูกที่สองเสียในช่วงนี้ ข้อมูลหาย. b2ctl จะไม่ยอมให้ offline ลูกที่สองระหว่างนี้
08

Burn-in — ทดสอบดิสก์ใหม่/มือสองก่อนใช้

ก่อนเชื่อใจดิสก์ใหม่ (โดยเฉพาะมือสอง) รันทดสอบเพื่อคัดตัวใกล้ตายออกก่อนเข้า pool — ได้ผลเป็น PASS / WARN / FAIL. เลือกหลายลูกพร้อมกันได้ และรันเบื้องหลัง

b2ctl> b
    [1] /dev/sdb (bay 32:4) Samsung SSD 870 EVO 1TB
    [2] /dev/sda (bay 32:5) Samsung SSD 870 EVO 1TB
  burn in which #> (space-separated) 1 2
  burn-in 2 disk(s) (long self-test)? [y/N]> y
  also run a full read-surface scan (badblocks, read-only, hours)? [y/N]> y

 BAY    DISK   SELF-TEST                    SURFACE SCAN (badblocks)
 32:4   sdb    [########------] 62%  ~1h10m  [###-----------] 18%  ~4h30m
 32:5   sda    [##########----] 74%  ~40m    [####----------] 22%  ~4h05m
แปลผล
  • PASS สะอาด ใช้ได้เลย
  • WARN ใช้ได้แต่เก่า (POH > 40,000 ชม., มี grown defect, หรือ scan เจอ bad block) → จัด priority ต่ำ
  • FAIL มี uncorrected error หรือ self-test ไม่ผ่าน → อย่าเอาเข้า pool
ปลอดภัย = อ่านอย่างเดียว: burn-in สั่งแค่ self-test และ (ถ้าเลือก) badblocks แบบ read-only — ไม่เขียนทับดิสก์. กด Ctrl-C เพื่อออกได้ โดยงานยังรันต่อเบื้องหลัง — กลับเข้ามาดูใหม่ด้วย b2ctl burnin --status หรือกด b อีกครั้ง
09

Dry-run — ซ้อมก่อนทำจริง

อยากดูว่า b2ctl จะรันคำสั่งอะไร โดยไม่แตะดิสก์จริง — เหมาะสำหรับฝึก, ตรวจสอบ, หรือเรียนรู้. เปิดได้ 2 วิธี

วิธีที่ 1 — กด t ในโหมด watch

b2ctl> t
[DRY-RUN] enabled — write commands will be printed, not executed
b2ctl> s
  swap (1:4) Samsung SSD 870 (…) onto spare (1:7)? [y/N]> y
  [DRY-RUN] would run: zpool replace tank \
    /dev/disk/by-id/ata-Samsung_SSD_870_EVO_1TB_S74ZNS0W... \
    /dev/disk/by-id/ata-Samsung_SSD_870_EVO_1TB_S74ZNS0W582283V

วิธีที่ 2 — ใส่ --dry-run ตั้งแต่เปิด

root@pve:~# b2ctl --dry-run watch     # หรือ b2ctl --dry-run swap
ขณะ dry-run ทำงาน
  • คำสั่ง เขียน (zpool, wipefs, sgdisk, dd) → แสดงเฉย ๆ ไม่รันจริง
  • คำสั่ง อ่าน (smartctl, zpool status) → ยังรันปกติ แสดงข้อมูลจริง
เคล็ดลับ: ใช้ dry-run ก่อนทุกครั้งที่ทำงานกับดิสก์ที่ไม่คุ้นเคย — เอาไว้ตรวจว่า path ถูกตัวก่อนกดยืนยันจริง
10

ระบบความปลอดภัย

b2ctl ออกแบบให้ "กดผิดก็ไม่พัง" — บันทึกทุก operation, snapshot ก่อนลงมือ, และย้อนกลับได้

กล่องยืนยันรายละเอียดครบ

ก่อนทุกคำสั่งที่เปลี่ยนแปลงดิสก์ ระบบแสดงกล่องยืนยันพร้อม path เต็มของดิสก์ และคำสั่งที่จะรัน:

┌─ CONFIRM OPERATION ────────────────────────────────┐
│ Op:    replace                                     │
│ From:  bay 1:4  S74ZNS0W...  ONLINE  (tank/raidz1) │
│ To:    bay 1:7  S8ABC...     AVAILABLE             │
│ Will run:                                          │
│   zpool replace tank /dev/disk/by-id/ata-...       │
│ Snapshot → /var/log/b2ctl/snapshots/2026...txt    │
└────────────────────────────────────────────────────┘
Proceed? [y/N]:
ค่าเริ่มต้นคือ N — กด Enter เปล่า ๆ = ยกเลิกทันที ไม่มีอะไรเกิดขึ้น

4 ชั้นความปลอดภัย

01Audit trail

ทุก operation บันทึกลง /var/log/b2ctl/ops.jsonl อัตโนมัติ — ดูย้อนหลังด้วย b2ctl log

02Snapshot ก่อนทำ

เก็บสถานะ pool + SMART ของดิสก์ก่อนลงมือ ไว้ที่ /var/log/b2ctl/snapshots/

03Rollback

ย้อน operation ที่ผ่านมาด้วย b2ctl rollback <op_id> (แสดง confirm ก่อน)

04Post-op verify

หลังทำเสร็จ scan pool ซ้ำ ถ้าผลไม่ตรงคาด จะเตือนพร้อมชี้ snapshot + คำสั่ง rollback

operation ไหนย้อนได้บ้าง

operationย้อนได้?
offline (ปิด disk)ได้
add spareได้
replaceได้ ถ้า resilver ยังไม่เสร็จ
demoteได้
create poolได้ แต่ต้อง destroy (ลบข้อมูล)
wipeไม่ได้ — ถาวร
11

หาตัวดิสก์จริงในแร็ค (locate)

ก่อนดึงดิสก์ออกทุกครั้ง ให้กะพริบไฟยืนยันตำแหน่งก่อน จะได้ไม่ดึงผิดตัว

b2ctl> l
  locate which (bay/serial/sdX)> 1:4
  blinking /dev/sdb for 5s ...
  ✔ done (via dd)

ระบุดิสก์ได้ 3 แบบ: เลข bay (1:4), ซีเรียล (S74ZNS0W…), หรือ ชื่อ device (sdc). จากนอกโหมด watch ก็สั่งได้ พร้อมกำหนดวินาที:

root@pve:~# b2ctl locate 1:4 30      # กะพริบ 30 วินาที เดินไปหาในแร็คได้สบาย
เคล็ดลับ: ใช้ locate ก่อนดึงดิสก์ออกทุกครั้ง — เพราะเลข bay เป็นแค่ป้ายแสดงผล ถ้าตั้งค่าสลับ อาจชี้ผิดช่องได้ แต่ไฟกะพริบไม่โกหก
ห้าม! อย่ากะพริบไฟ (locate) บนดิสก์ที่กำลัง resilver อยู่ — รอให้ resilver เสร็จก่อน (เห็น resilvered ... with 0 errors) แล้วค่อย locate
12

ข้อควรระวัง

อย่าผสม SAS กับ SATA โดยไม่ทดสอบก่อน

เอาดิสก์ SAS มาเป็น spare ในพูลที่เป็น SATA ล้วน อาจมีปัญหา ถ้าไม่แน่ใจ ให้ใช้ดิสก์ชนิดเดียวกับที่มีอยู่ใน pool

หมายเลข Bay เป็นแค่ตัวแสดง

บน Dell R620 ที่แฟลชเป็น IT mode หมายเลข slot อาจสลับกัน. ทุกการทำงานของ b2ctl ใช้ ซีเรียล ของดิสก์ ไม่ใช่เลข bay ดังนั้นถ้า bay ดูผิด ให้ใช้ l (locate) กะพริบไฟยืนยันก่อนเสมอ

ทุกคำสั่งที่เปลี่ยนข้อมูลจะถามยืนยัน

ไม่ต้องกลัวกดผิดแล้วข้อมูลหาย — ระบบแสดงกล่อง Confirm พร้อม path เต็มก่อนทุกครั้ง ค่าเริ่มต้นคือ N (No) กด Enter เปล่า ๆ จะไม่มีอะไรเกิดขึ้น

ZFS resilver ใช้เวลา — ห้ามขัดจังหวะ

การ resilver (สร้างข้อมูลใหม่) ใช้เวลาตามปริมาณข้อมูล ห้ามปิดเครื่องหรือถอดดิสก์ระหว่าง resilver เพราะช่วงนั้น pool ยังไม่มี redundancy เต็ม

เมื่อไม่แน่ใจ: กด s (skip) ไว้ก่อนเสมอ — ดิสก์จะไม่ถูกเปลี่ยนแปลง แล้วค่อยกลับมาจัดการทีหลังด้วย a (assign) หรือเปิด dry-run (t) ซ้อมดูก่อน

คำสั่ง CLI (พิมพ์จาก terminal)

คำสั่งทำอะไร

b2ctl status --jsonแสดงผลเป็น JSON
b2ctl watch⭐ เข้าโหมดเฝ้าดู (แนะนำ)
b2ctl --dry-run watchเข้าโหมดเฝ้าดูแบบ dry-run
b2ctl locate <bay/serial/sdX> [วินาที]กะพริบไฟหาดิสก์
b2ctl burnin <dev …> [--scan]ทดสอบดิสก์ก่อนใช้ (หลายลูกได้)
b2ctl burnin --statusกลับเข้าดู burn-in ที่รันอยู่
b2ctl checkตรวจสภาพระบบ / tool ครบไหม
b2ctl log [--last N]ดู operation ล่าสุดจาก audit trail
b2ctl rollback <op_id>ย้อน operation ก่อนหน้า (มี confirm)
b2ctl versionแสดงเวอร์ชัน
